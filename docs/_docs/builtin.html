fn realloc_data #

unsafe
fn realloc_data(old_data &byte, old_size int, new_size int) &byte

realloc_data resizes the memory block pointed by old_data to new_size bytes. old_data must be a pointer to an existing memory block, previously allocated with malloc, v_calloc or vcalloc, of size old_data. realloc_data returns a pointer to the new location of the block. NB: if you know the old data size, it is preferable to call realloc_data, instead of v_realloc, at least during development, because realloc_data can make debugging easier, when you compile your program with -d debug_realloc.
